## Value Formula

**Adjusted Value = ScaleFactor × ((RawVal − Floor)^Exponent) / ((10000 − Floor)^Exponent)**

### Dynasty defaults
| Parameter | Default | Notes |
|---|---|---|
| Player Floor | 2500 | Assets below this = 0 value |
| Pick Floor | 2400 | Slightly lower than player floor |
| Scale Factor | 10000 | Top asset always equals this |
| Exponent | 1.25 | >1 compresses depth |
| Max Raw Val | 10000 | Fixed KTC maximum |

### Redraft defaults
| Parameter | Default |
|---|---|
| Redraft Floor | 4000 |
| Redraft Exponent | 1.25 |

Formula inputs update all values instantly — no sync needed. Per-league overrides (floor/scale/exponent only) stored in `localStorage` under `dynastyCalc_leagueOverrides_v1`.

### Value Source Registry
Phase 1 of a multi-source value architecture. All player valuation in the app routes through a single registry rather than hardcoded KTC reads.

**Key functions:**
- `registerValueSource(def)` — adds source to `VALUE_SOURCES`; each source: `{ id, label, type, getRaw(name, pos, sf, tep) }`
- `getActiveSource()` — returns currently selected source (falls back to `'ktc'`)
- `setActiveValueSource(id)` — persists to localStorage, re-renders rankings/leagues/calc
- `getRawValue(name, pos, ligId)` — dispatches through `getActiveSource().getRaw(...)` — **single entry point for all valuation**
- `getRawKTC(name, pos)` — **always** reads from `VALUE_SOURCES['ktc']`; used for raw-value badge display
- `renderValueSourceSelector()` — renders pill buttons into `#value-source-selector` in Config tab

**Currently registered sources:**
| ID | Label | Type | Notes |
|---|---|---|---|
| `ktc` | KTC Dynasty | `ktc` | Default; reads from `allPlayers` / `ktcMap` |

### Name Matching
FFPC and KTC name formats diverge. Handled by:
- `normalizeName(n)` — strips suffixes (Jr/Sr/II/III/IV), periods, extra spaces
- `PLAYER_ALIASES` map — e.g. `'chig okonkwo' → 'chigoziem okonkwo'`
- `getKTCPlayer(name)` — tries direct map, then normalized map, then partial match
- Rostered set built AFTER KTC lookup to avoid FA false-positives

### Key Functions
- `adjVal(name, pos, ligId)` — adjusted dynasty value using per-league params
- `adjValue(raw, player)` — adjusted dynasty value using global params (trade calc)
- `adjRedraft(name, pos)` — adjusted redraft value
- `syncAll()` — chains KTC dynasty + redraft scrape + FFPC sync + player map build
- `swCheckUpdate()` — (app.html only) forces SW update check + reload
- `openPlayerCard(name)` — opens global player card modal
- `buildPCValueHistorySection(name)` — renders dynasty value history SVG chart
- `buildPCScoringSection(name)` — renders PPG bar charts + game-by-game breakdown
- `loadKtcHistory()` / `loadPlayerStats()` / `loadTeamHistory()` — load from localStorage or proxy

## KTC Value History System

### Data Structure (`ktc-history.json`)
```json
{
  "2025-10-07": {
    "Josh Allen": { "value": 9995, "oneqb": 7651, "tep": 9995, "pos": "QB", "team": "BUF", "age": 29.9 },
    ...
  },
  "2026-04-04": { ... }
}
```

### Daily Snapshot Flow
- `snapshotDate(dateStr, players)` — writes current cached KTC values into `ktcHistory[dateStr]`
- `fillMissedSnapshots(players)` — called on boot; fills any gaps since last snapshot
- Nightly 2am cron via `setInterval` in proxy
- Saved to `ktc-history.json` on disk

### Backfill System (NEW — no longer uses Puppeteer)
KTC exposes historical value data via `POST /dynasty-rankings/histories` accepting an array of `{playerID: N}` objects.

**Endpoint discovery:**
- URL format: `POST https://keeptradecut.com/dynasty-rankings/histories`
- Body: `[{playerID: 365}, {playerID: 1414}, ...]` (up to ~50 per request)
- Response: `[{playerID: 1414, oneQB: {valueHistory: [...]}, superflexValues: {valueHistory: [...]}, ...}]`

**Encoding format:** Each history entry is a 10-character string: `YYMMDDVVVV`
- `YY` = 2-digit year (e.g. `25` = 2025)
- `MM` = 2-digit month
- `DD` = 2-digit day
- `VVVV` = 4-digit value (0000–9999, matches KTC's 0–10000 scale)
- Example: `"2510079995"` → date `2025-10-07`, value `9995`

**Backfill implementation in `ktc-proxy.js`:**
- `ktcHistoriesPost(playerIDs)` — POSTs batch of IDs, returns parsed JSON
- `decodeKTCHistoryEntry(encoded)` — decodes YYMMDDVVVV → `{dateStr, value}`
- `backfillKTCHistory()` — batches 548 players in groups of 50, ~11 requests, ~17 seconds total (vs. ~45 min with old Puppeteer approach)
- Triggered via `POST /ktc-history/backfill`
- Progress via `GET /ktc-history/backfill-status`

**PENDING WORK:**
- Store all format variants in backfill: `superflexValues` (SF dynasty), `oneQB` (1QB dynasty), redraft equivalents, and all TEP tiers
- Allow player card history chart to swap between formats based on league context or user selection
- Ensure TE scoring history is TEP-adjusted per league format
- Label "KTC Raw" clearly as dynasty or redraft depending on context

### KTC Player ID Format
Players in `ktc-cache.json` have a numeric `id` field (string). URL slug for player pages:
```
{name-slug}-{id}  →  josh-allen-365, bijan-robinson-1414
```
Slug formula: `name.toLowerCase().replace(/['']/g,'').replace(/[^a-z0-9\s-]/g,'').replace(/\s+/g,'-') + '-' + id`

## Known Quirks & Decisions

- **KTC backfill no longer uses Puppeteer** — replaced with direct `POST /dynasty-rankings/histories` which returns compressed history for 50 players per request. ~17 seconds for all 548 players. Encoded as `YYMMDDVVVV` strings.
- **KTC player IDs** — numeric strings in `ktc-cache.json` (`id` field). Player page URL: `{slug}-{id}` (e.g. `josh-allen-365`). The old format (slug only) 404s.
- **History chart zero filtering** — `buildPCValueHistorySection` skips entries where `value <= 0` to prevent false spikes from missing data.
- **"101 snapshots" display** — if backfill has run, `pts.length` reflects actual history dates for that player. Chart shows flat-line spike when only 1 date exists (today's snapshot) — guard `pts.length < 2` shows "No history yet" instead.
- **Value source architecture** — `getRawValue()` dispatches through `getActiveSource().getRaw()`. `getRawKTC()` is pinned to KTC source so raw-value badges always show KTC numbers. Adding a new source only requires `registerValueSource({...})`.
- **FFPC + KTC name divergence** — suffix stripping + `PLAYER_ALIASES` map; rostered set built after KTC lookup to avoid FA false-positives.
- **All FFPC leagues are SF + TE+** — hardcoded, not overridable via league settings.
- **Pick slot projection** — uses redraft ranking + dynasty value composite to estimate Early/Mid/Late, then interpolates KTC tier anchor values.
- **iOS PWA header** — `viewport-fit=cover` + `padding-top: env(safe-area-inset-top)` on `.page-tabs` + `::before` fills notch area.
- **app.html config is default tab** — `view-config` has `active` class in static HTML; `view-calc` does NOT.
- **SW update** — `skipWaiting()` in SW install; `swCheckUpdate()` in app.html calls `registration.update()` then listens for `updatefound` and reloads.
- **build_app.py must be run** after any `index.html` change; push `app.html` + `sw-app.js` to GitHub.
- **Dynasty Nerds JWT** — stored in `ffpc-config.json` as `dynJwt`, expires periodically; update when Scout stops working.
- **PC browser proxy trust** — visit `https://100.85.44.43:3001/trust` in Edge on PC, or navigate to `https://100.85.44.43:3001/ktc` and click through cert warning once. Phone works without this step.

2. `ktc-proxy.js` changed → kill and restart proxy: `pkill -f "node ktc-proxy.js" && node ktc-proxy.js &`
3. After proxy restart → trigger backfill if needed: `curl -k -X POST https://localhost:3001/ktc-history/backfill`
4. Check backfill progress: `curl -k https://localhost:3001/ktc-history/backfill-status`
