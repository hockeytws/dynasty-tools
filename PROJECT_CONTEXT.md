## Value Formula

**Adjusted Value = ScaleFactor × ((RawVal − Floor)^Exponent) / ((10000 − Floor)^Exponent)**

### Dynasty defaults
| Parameter | Default | Notes |
|---|---|---|
| Player Floor | 2500 | Assets below this = 0 value |
| Pick Floor | 2400 | Slightly lower than player floor |
| Scale Factor | 10000 | Top asset always equals this |
| Exponent | 1.25 | >1 compresses depth |
| Max Raw Val | 10000 | Fixed KTC maximum |

### Redraft defaults
| Parameter | Default |
|---|---|
| Redraft Floor | 4000 |
| Redraft Exponent | 1.25 |

Formula inputs update all values instantly — no sync needed. Per-league overrides (floor/scale/exponent only) stored in `localStorage` under `dynastyCalc_leagueOverrides_v1`.

### Adjusted Value Functions
- `adjDyn(raw)` / `adjValue(raw, player)` — dynasty adjusted value (trade calc uses global params)
- `adjVal(name, pos, ligId)` — dynasty adjusted value (league views use per-league overrides)
- `adjRedraftValue(name, pos)` / `adjRedraft(raw)` — redraft adjusted value with separate floor/exponent

## KTC Value History System

### Data Structure (`ktc-history.json`, ~13MB, ~181 days)
```json
{
  "2025-10-08": {
    "Brock Bowers": {
      "value": 7635, "oneqb": 8197,
      "tep": 8455, "tepp": 9369, "teppp": 10139,
      "tep_1qb": 9072, "tepp_1qb": 9647, "teppp_1qb": 10432,
      "pos": "TE", "team": "LVR", "age": 23.3
    }
  }
}
```

### Daily Snapshot Flow
- `snapshotDate(dateStr, players)` — writes current cached KTC values into `ktcHistory[dateStr]`
- `fillMissedSnapshots(players)` — called on boot; fills any gaps since last snapshot
- Nightly 2am cron via `setInterval` in proxy
- Daily snapshots now include picks (RDP)
- Saved to `ktc-history.json` on disk
- `tepVal()` helper uses `'value' in obj` check (not `&&` + `||`) to handle 0 values correctly

### Backfill System (API-based, all formats)
KTC exposes historical value data via `POST /dynasty-rankings/histories`.

**API details:**
- URL: `POST https://keeptradecut.com/dynasty-rankings/histories`
- Body: `[{playerID: 365}, {playerID: 1414}, ...]` (up to ~50 per request)
- Response: `[{playerID: 1414, oneQB: {valueHistory: [...]}, superflexValues: {valueHistory: [...], tep: {valueHistory: [...]}, tepp: {...}, teppp: {...}}, ...}]`

**CRITICAL API field names (discovered via debugging):**
- Response uses `entry.superflexValues` (NOT `entry.superflexValues` — the older doc said `superflexValues` but it was actually `entry.superflex` at one point; current correct key is `entry.superflexValues`)
- SF TEP history: `sf.tep.valueHistory` (tep is an OBJECT with `{value, rank, positionalRank, history}`)
- 1QB TEP: `qb1.tep.valueHistory`
- `parseAllHistories(entry)` handles all format variants with fallback key names

**Encoding format:** Each history entry is a 10-character string: `YYMMDDVVVV`
- `YY` = 2-digit year, `MM` = month, `DD` = day, `VVVV` = 4-digit value (0000–9999)
- Example: `"2510079995"` → date `2025-10-07`, value `9995`

**Backfill implementation:**
- `ktcHistoriesPost(playerIDs)` — POSTs batch of IDs, returns parsed JSON
- `decodeKTCHistoryEntry(encoded)` — decodes YYMMDDVVVV → `{dateStr, value}`
- `backfillKTCHistory()` — batches all non-RDP players in groups of 50, ~17 seconds total
- Chains redraft backfill: `.then(() => backfillRedraftHistory())`
- Post-backfill verification: checks TE player has distinct SF/TEP values
- Startup self-heal: if today's snapshot has SF=0 for TEs, auto-deletes and re-snapshots

### KTC Redraft History System
Separate file `ktc-redraft-history.json` (~4MB, ~95 days). Same date→player→values structure.

**Key differences from dynasty history:**
- Uses `POST /fantasy-rankings/histories` endpoint (NOT dynasty-rankings)
- Returns ALL ~377 players in one call regardless of input IDs
- Redraft player IDs differ from dynasty IDs (Bijan = 1507 redraft, 1414 dynasty, only ~129/300 overlap)
- Fixed with name-based fallback lookup + `processedNames` Set to prevent duplicate overwrites (API returns both redraft ID and dynasty ID entries for same player — second entry would overwrite first with empty data)

### KTC Page Structure (Puppeteer scrape)
- `window.playersArray[].superflexValues.value` = SF dynasty value (number)
- `window.playersArray[].superflexValues.tep` = object `{value, rank, positionalRank, history}` (NOT a number!)
- `tepVal()` helper: uses `'value' in obj` check because `sf.tep && sf.tep.value || sf.value` fails when `sf.tep.value` is 0 (falsy)

### KTC Player ID Format
Players in `ktc-cache.json` have a numeric `id` field (string). URL slug for player pages:
```
{name-slug}-{id}  →  josh-allen-365, bijan-robinson-1414
```

## Known Quirks & Decisions

### KTC Data Pipeline
- **KTC page `superflexValues.tep`** is an OBJECT `{value, rank, positionalRank, history}`, NOT a number — use `'value' in obj` check
- **JS operator precedence trap:** `a && a.value || fallback` fails when `a.value` is 0 (falsy) — always use explicit `'value' in obj` check via `tepVal()` helper
- **KTC API `entry.superflex`** (not `entry.superflexValues`) with `sf.tepHistory` (not `sf.tep.valueHistory`) — key names differ between page scrape and API
- **Redraft API returns ALL players** regardless of input IDs, and uses different player IDs than dynasty
- **Duplicate API entries:** API returns both redraft ID (1507) and dynasty ID (1414) for same player — second entry overwrites first with empty data. Fixed with `processedNames` Set.

### Infrastructure & Deployment
- **`fuser -k 3001/tcp`** is more reliable than `pkill -f "node ktc-proxy.js"` for killing the proxy
- **FFPC player map:** Must reload from disk when file is newer than in-memory cache to avoid requiring proxy restarts
- **iOS PWA deployment:** GitHub Pages takes 1–5 minutes to deploy; service worker absolute paths break in subdirectory deployments; iOS requires manual Safari visit to install updated service workers
- **Trade calc initialization:** `buildCalcUI()` must not run before `allPlayers` is ready — fixed with polling interval
- **Offline sync:** Use a 5-second HEAD request reachability check before attempting sync to avoid indefinite spinner
- **localStorage 5MB limit** — dynasty history alone is 13MB. Use IndexedDB for PWA (`app.html`).
- **`<script>` tags in innerHTML** don't execute — use `data-*` attributes instead

### UI & Rendering
- **History chart shows ADJUSTED values** (not raw KTC). Uses `adjDyn()` for dynasty, `adjRd()` for redraft with separate floors/exponents.
- **History zero filtering** — `buildPCValueHistorySection` skips entries where `value <= 0` to prevent false spikes from missing data.
- **"No history yet"** — guard `pts.length < 2` shows this instead of flat-line spike when only 1 date exists.
- **Scoring charts label "Half PPR"** — NO fake TEP adjustment applied since Player Profiler data is not TEP-adjusted.
- **PROFILES default sort = DYNASTY** (not PPG). Button order: DYNASTY, REDRAFT, PPG. PPG sort uses career PPG (`getCareerPPG()` — weighted average across all seasons). PICKS filter shows picks in ALL view. Pick rows show dynasty values, labeled "PICK". Picks are clickable and open player cards.
- **Date format with year:** Player card value history X-axis: `Oct 8 '25`. Team card chart: same. Transaction dates: `Apr 5, 2026`. Slider tooltip: `Oct 8, 2025`.
- **Format switching freeze fix:** `_pcHistUpdating` boolean lock prevents rapid clicks from queuing renders.
- **Value source architecture** — `getRawValue()` dispatches through `getActiveSource().getRaw()`. `getRawKTC()` is pinned to KTC source.
- **FFPC + KTC name divergence** — suffix stripping + `PLAYER_ALIASES` map; rostered set built after KTC lookup.
- **Pick slot projection** — uses redraft ranking + dynasty value composite to estimate Early/Mid/Late, then interpolates KTC tier anchor values. Mid-round estimates (slot 6.5) preferred over slot projections.
- **iOS PWA header** — `viewport-fit=cover` + `padding-top: env(safe-area-inset-top)` on `.page-tabs`.
- **app.html config is default tab** — `view-config` has `active` class in static HTML.
- **SW update** — `skipWaiting()` in SW install; APP UPDATE button in offline app's Config page calls `registration.update()`.
- **Dynasty Nerds JWT** — stored in `ffpc-config.json` as `dynJwt`, expires periodically; update when Scout stops working.

### FFPC Data Quirks
- Player ID mapping uses `playerID=` (not `pid=`) in hrefs
- Rosters page uses `ltuid=` (not `leagueID=`) as URL parameter
- FFPC strips name suffixes (Jr./Sr./III), omits periods in initials (DJ Moore vs D.J. Moore), and uses nicknames — requires `normalizeName()` and `PLAYER_ALIASES` map

### DN Pick Data Quirk
- DN reuses the same `pickId` per round across all teams (e.g., 421=R1, 422=R2), so deduplication by `year-round-pickId` collapses multiple picks — raw array must be trusted without dedup
- `previousTeamId` always equals current team ID so traded-from attribution is unavailable
